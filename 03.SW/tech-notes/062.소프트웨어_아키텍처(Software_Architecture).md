# 소프트웨어 아키텍처(Software Architecture)

> [!info] **소프트웨어 아키텍처란?**
> 소프트웨어 시스템의 구성 요소(Component), 요소 간의 관계(Relationship), 그리고 설계 및 진화를 통제하는 원칙(Principle)과 가이드라인의 집합입니다. 시스템의 뼈대를 결정하며, 비기능적 요구사항(품질 속성)을 해결하는 핵심 수단입니다.

---

## 1. 소프트웨어 아키텍처의 개요
### 가. 소프트웨어 아키텍처의 정의
- 시스템의 기본 구조를 형성하는 **컴포넌트**와 이들의 **외부 인터페이스**, 그리고 컴포넌트 간의 **상호작용**을 정의한 설계도

### 나. 아키텍처의 필요성 (Why)
1. **복잡도 관리**: 거대해지는 시스템을 추상화하여 이해하고 관리할 수 있게 함
2. **이해관계자 의사소통**: 고객, 개발자, 관리자 간의 공통된 이해와 합의의 기반(Baseline) 제공
3. **품질 속성 달성**: 성능, 보안성, 가용성 등 비기능 요구사항을 설계 단계에서 보장
4. **재사용성 및 유지보수성**: 검증된 패턴 적용을 통해 개발 생산성 향상 및 변경 영향 최소화

---

## 2. 아키텍처 모델링 및 주요 패턴 (What & How)
### 가. 아키텍처 4+1 View (Kruchten 모델)
- 이해관계자별 관점을 반영하기 위한 5가지 뷰로 구성

| 뷰 (View) | 관점 및 역할 | 대상 (Stakeholder) |
| :--- | :--- | :--- |
| **Logical View** | 기능적 요구사항(클래스, 객체 관계) | 설계자, 비즈니스 분석가 |
| **Process View** | 비기능 요구사항(성능, 병렬 처리, 확장성) | 시스템 통합자 |
| **Development View** | 구현 관점(컴포넌트, 패키지, 레이어) | 개발자, 소프트웨어 관리자 |
| **Physical View** | 배포 및 실행 환경(하드웨어 매핑, 네트워크) | 시스템 엔지니어 |
| **Use Case View** | 타 뷰의 도출 근거가 되는 시나리오 (필수) | 모든 이해관계자 |

### 나. 주요 아키텍처 스타일 및 패턴
- **계층화 패턴(Layered Pattern)**: 관심사 분리(SoC)를 통한 수평적 계층 구조 (N-Tier)
- **마이크로서비스(MSA)**: 서비스 단위의 독립적 배포 및 확장 (Polyglot)
- **이벤트 기반(Event-Driven)**: 비동기 메시징을 통한 느슨한 결합(Loose Coupling)
- **헥사고날(Hexagonal)**: 비즈니스 로직과 외부 인터페이스(DB, UI)의 철저한 분리

---

## 3. 아키텍처 품질 속성과 평가 방법
### 가. 품질 속성 시나리오 (Quality Attribute Scenario)
- "자극원 → 자극 → 환경 → 대상 → 응답 → 응답 측정"의 구조로 품질 요구사항을 구체화

### 나. 아키텍처 평가 방법론 비교 분석

| 방법론 | 특징 | 평가 관점 |
| :--- | :--- | :--- |
| **SAAM** | 최초의 아키텍처 평가 방법 | 수정 용이성(Modifiability) 중심 |
| **ATAM** | 가장 널리 사용되는 방법론 | 여러 품질 속성 간의 **절충(Trade-off)** 분석 |
| **CBAM** | 경제적 관점 추가 | 비용 대비 이익(ROI) 분석을 통한 의사결정 |
| **ADR** | 아키텍처 설계 검토 | 설계 결정 사항의 타당성 및 리스크 검토 |

---

## 4. 기술사적 제언 및 실무 적용 방안
### 가. 아키텍처 설계 시 고려사항
1. **Trade-off 분석의 중요성**: 모든 품질 속성을 동시에 극대화할 수 없으므로(예: 보안성 강화 시 성능 저하), 비즈니스 우선순위에 따른 균형점이 필수적임
2. **Technical Debt 관리**: 단기적 개발 속도를 위해 아키텍처 원칙을 위배할 경우, 지속적인 리팩토링을 통해 아키텍처 부패를 방지해야 함

### 나. 최신 IT 트렌드와 연계한 제언
- **Cloud Native Architecture**: 서버리스, 컨테이너 환경에 최적화된 복원력(Resilience)과 확장성(Elasticity) 중심의 아키텍처 설계가 요구됨
- **AI-Driven Architecture**: AI 모델의 서빙과 데이터 파이프라인을 효율적으로 처리하기 위한 **Mloas** 아키텍처와의 통합이 가속화되고 있음
