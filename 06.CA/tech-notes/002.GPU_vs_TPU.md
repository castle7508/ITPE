# 1. GPU와 TPU의 개념 및 특징

## 가. GPU (Graphics Processing Unit)
- **개념**: 수천 개의 코어로 구성되어 병렬 처리에 특화된 범용 그래픽 처리 장치
- **핵심 기술 (CUDA)**: NVIDIA의 병렬 컴퓨팅 플랫폼 및 API 모델. C/C++ 기반으로 GPU 리소스를 제어하여 딥러닝 연산을 가속화함
- **특징**: 높은 유연성(Programmability)을 바탕으로 그래픽 처리뿐만 아니라 일반 목적의 병렬 연산(GPGPU)에 널리 활용

## 나. TPU (Tensor Processing Unit)
- **개념**: 구글(Google)이 딥러닝 연산, 특히 **텐서(Tensor)** 계산을 최적화하기 위해 설계한 맞춤형 주문형 반도체(ASIC)
- **핵심 기술 (TensorFlow)**: 구글의 오픈소스 머신러닝 라이브러리. TPU는 TensorFlow 프레임워크와 밀접하게 결합되어 대규모 행렬 연산을 극도로 고속화함
- **특징**: **Systolic Array** 아키텍처를 사용하여 데이터 이동 시 전력 소모를 최소화하고 연산 처리량(Throughput)을 극대화

# 2. GPU와 TPU의 비교 분석

| 비교 항목 | GPU (Graphics Processing Unit) | TPU (Tensor Processing Unit) |
|---|---|---|---|
| **설계 목적** | 범용 병렬 처리 (Graphics, AI, HPC) | 딥러닝 특화 가속 (Tensor 연산) |
| **아키텍처** | SIMT (Single Instruction Multiple Threads) | **Systolic Array** (Matrix Multiplier) |
| **유연성** | 매우 높음 (다양한 알고리즘 구현 가능) | 상대적으로 낮음 (행렬 연산에 최적화) |
| **정밀도** | FP32, FP64 등 고정밀 연산 강점 | bfloat16, INT8 등 저정밀 고효율 연산 |
| **주요 벤더** | NVIDIA (A100, H100) | Google (Cloud TPU v4, v5) |
| **개발 환경** | CUDA, OpenCL | TensorFlow, JAX, PyTorch(XLA) |

# 3. 최근 GPU보다 TPU를 사용하는 주요 이유

1.  **행렬 연산 최적화**: 트랜스포머(Transformer) 기반의 LLM(대형 언어 모델)은 방대한 행렬 곱셈 연산이 핵심이며, TPU는 이를 처리하기 위한 전용 하드웨어 로직을 갖춤
2.  **데이터 이동 최소화**: TPU의 시스톨릭 어레이 구조는 연산 중간 결과를 메모리에 쓰지 않고 다음 연산기로 직접 전달하여 **메모리 병목 현상(Von Neumann Bottleneck)**을 획기적으로 해결
3.  **학습 비용 효율성**: 클라우드 환경(GCP)에서 대규모 모델 학습 시, 동일 성능 대비 전력 소모와 학습 시간 단축을 통해 총소유비용(TCO) 절감 가능
4.  **분산 학습 확장성**: TPU Pod 아키텍처는 수천 개의 TPU 칩을 고속 인터커넥트로 연결하여 선형적인 성능 확장이 가능함

# 4. 효율성 측면에서 TPU의 장점 및 향후 전망

## 가. 효율성 측면의 장점
- **에너지 효율(Performance per Watt)**: 불필요한 범용 제어 로직을 제거하고 연산 유닛에 집중하여 와트당 성능이 GPU 대비 우수
- **학습 처리량(Throughput)**: 저정밀도 연산(bfloat16) 기술을 선도적으로 적용하여 데이터 처리 속도를 극대화
- **소프트웨어 수직 계열화**: TensorFlow/XLA 컴파일러와 하드웨어의 긴밀한 통합을 통해 최적화된 실행 계획 수립

## 나. 향후 전망
- **맞춤형 ASIC 경쟁 가속**: 구글뿐만 아니라 AWS(Trainium), MS(Maia) 등 빅테크 기업들의 자체 AI 반도체 개발 가속화
- **엣지 인프라 확장**: 클라우드용 TPU뿐만 아니라 모바일 및 IoT 기기용 **Edge TPU** 보급을 통한 온디바이스 AI 활성화
- **소프트웨어 생태계 범용화**: XLA(Accelerated Linear Algebra) 기술의 발전으로 특정 프레임워크 종속성을 탈피하고 PyTorch 등 타 라이브러리와의 호환성 지속 강화

> [!tip] **기술사 인사이트**
> GPU가 '다양한 요리를 할 수 있는 다목적 주방'이라면, TPU는 '특정 메뉴(행렬 연산)를 대량으로 찍어내는 자동화 공장'입니다. LLM 시대를 맞아 **범용성보다는 효율성**이 중시되면서, 기업들의 선택은 자체 가속기(ASIC)인 TPU 계열로 이동하고 있습니다.
