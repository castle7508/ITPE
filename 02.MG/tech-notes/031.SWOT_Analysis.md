# 1. SWOT 분석(SWOT Analysis)의 개요

### 가. SWOT 분석의 정의
- 기업의 내부 환경 요소인 **강점(Strength)**과 **약점(Weakness)**, 외부 환경 요소인 **기회(Opportunity)**와 **위협(Threat)**을 분석하여 전략적 대안을 도출하는 기법임
- 내부 역량 분석과 외부 환경 분석을 결합하여 현재의 위치를 파악하고 미래 전략을 수립하는 데 활용됨

### 나. 등장 배경 및 필요성
- **전략적 정렬(Strategic Alignment)**: 기업의 내부 역량과 외부 환경 간의 최적 조합을 찾아내어 비즈니스 목표 달성 가능성 제고
- **객관적 자기 진단**: 감성적 판단이 아닌 데이터 기반의 강/약점 파악을 통해 현실적인 사업 계획 수립
- **전략적 대안 도출**: 단순히 현상을 분석하는 데 그치지 않고, 4가지 조합을 통해 구체적인 실행 전략(SO, ST, WO, WT) 도출

# 2. SWOT 분석의 아키텍처 및 매트릭스 구성

### 가. SWOT 분석 매트릭스 [두음: 강약기위]
```mermaid
grid-layout
    S["강점 (Strength)"]
    W["약점 (Weakness)"]
    O["기회 (Opportunity)"]
    T["위협 (Threat)"]
```
*(실제 분석 시 아래와 같은 2x2 매트릭스로 활용)*

| | **내부 강점 (Strength)** | **내부 약점 (Weakness)** |
| :--- | :--- | :--- |
| **외부 기회 (Opportunity)** | **SO 전략 (공격적 전략)**: 강점을 활용하여 기회를 포착 | **WO 전략 (방향전환 전략)**: 약점을 보완하여 기회를 활용 |
| **외부 위협 (Threat)** | **ST 전략 (다각화 전략)**: 강점을 활용하여 위협을 회피 | **WT 전략 (방어적 전략)**: 약점을 보완하고 위협을 최소화 |

### 나. SWOT 분석의 4대 전략 [두음: 정공혁방]
| 전략 유형 | 특징 | 상세 설명 |
| :--- | :--- | :--- |
| **SO 전략 (적극적/공격적)** | 기회 활용 + 강점 극대화 | 시장 점유율 확대, 신규 시장 진입, 기술 선도 |
| **ST 전략 (혁신적/다각화)** | 위협 회피 + 강점 활용 | 신제품 개발, 시장 다변화, 위협 요소의 내부 역량 대응 |
| **WO 전략 (방향전환/보완)** | 기회 활용 + 약점 보완 | 전략적 제휴, 핵심 기술 도입, 조직 역량 강화 |
| **WT 전략 (방어적/철수)** | 위협 최소화 + 약점 보완 | 원가 절감, 사업 축소 또는 철수, 집중화 전략 |

# 3. SWOT 분석의 절차 및 심화 분석

### 가. SWOT 분석 수행 3단계 절차
1) **환경 분석 (Scanning)**: PEST 분석(외부) 및 가치사슬 분석(내부)을 통해 S, W, O, T 요소를 리스트업
2) **SWOT 매트릭스 작성 (Mapping)**: 각 요소 간의 연관성을 고려하여 2x2 매트릭스에 배치
3) **전략 대안 도출 (Formulating)**: 교차 분석을 통해 SO, ST, WO, WT 관점의 구체적인 전략 과제 도출

### 나. SWOT 분석의 한계 및 보완 (TOWS 분석)
- **한계**: 정적인 분석에 치우치기 쉬우며, 각 요소의 우선순위나 가중치를 반영하기 어려움
- **보완 (TOWS)**: SWOT이 현상 파악(Input) 중심이라면, TOWS는 도출된 전략(Output)과 실행 방안에 더 집중하는 역순 분석 방식으로 활용됨

# 4. 기술사적 제언 및 실무 적용 방안

### 가. 실무 도입 시 고려사항
- **요소의 객관성**: "강점"이라고 판단한 항목이 경쟁사 대비 실제 우위가 있는지 **VRIO 프레임워크** 등을 연계하여 철저히 검증해야 함
- **전략의 우선순위**: 도출된 수많은 전략 과제 중 리소스와 시급성을 고려하여 실행 우선순위를 결정하는 거버넌스 체계 필요

### 나. 보안(Security) 및 거버넌스 통제 방안
- **위협(T) 기반 대응**: 외부의 사이버 위협 환경(Threat)을 SWOT 분석에 반영하여, 내부 보안 역량(S, W)과 결합한 전사적 보안 강화 전략 수립
- **보안 거버넌스 정렬**: WT 전략 관점에서 내부 보안 취약점(W)을 개선하고 외부 컴플라이언스 위험(T)을 최소화하는 방어 전략 수립

### 다. 발전 방향 및 제언
- **데이터 기반 SWOT (D-SWOT)**: 빅데이터 분석과 소셜 리스닝을 통해 외부 기회/위협 요소를 실시간으로 포착하고 전략에 반영하는 동적 분석 체계로 진화
- **Digital Transformation과의 결합**: 기술적 강점(S)을 클라우드 및 AI로 전환하여 외부 디지털 시장의 기회(O)를 선점하는 DX SWOT 전략 필수

> [!tip] **기술사 인사이트**
> SWOT 분석은 단순한 나열이 아니라 **"전략적 인과관계"**를 찾아내는 과정입니다. 기술사 시험에서 SWOT을 활용할 때는 단순 4칸 채우기를 넘어, 도출된 전략들이 어떻게 비즈니스 목표와 기술적 해결책으로 연결되는지 논리적 스레드를 보여주는 것이 중요합니다.
