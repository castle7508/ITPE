# 1. 오류 제어 (Error Control)의 개요

## 가. 정의
- 데이터 전송 시 잡음, 감쇠, 간섭 등으로 발생하는 데이터 변형(Error)을 탐지하고 수정하여 신뢰성 있는 통신을 보장하는 기술
- **흐름 제어(Flow Control)**와 함께 데이터 링크 계층(2계층) 및 전송 계층(4계층)의 핵심 기능임

## 나. 오류 제어의 두 가지 측면
1.  **오류 검출 (Error Detection)**: 수신 측에서 데이터의 변형 여부만을 확인하는 과정 (예: Parity, Checksum, CRC)
2.  **오류 정정 (Error Correction)**: 오류를 탐지함과 동시에 원래의 데이터로 복원하는 과정 (예: Hamming Code, Reed-Solomon)

# 2. 오류 검출과 오류 정정의 비교

| 비교 항목 | 오류 검출 (Error Detection) | 오류 정정 (Error Correction) |
|---|---|---|
| **핵심 목적** | 오류 발생 여부 판별 | 오류 위치 파악 및 복구 |
| **방식** | 잉여 비트(Redundancy) 추가 | 전진 오류 정정(FEC) / 후진 오류 정정(BEC) |
| **대표 기술** | Parity Check, CRC, Checksum | Hamming Code, Turbo Code |
| **처리 효율** | 부하가 적고 구현이 단순함 | 부하가 크고 복잡한 연산 필요 |
| **주요 활용** | 유선 통신, 신뢰성 높은 매체 | 무선 통신, 위성 통신 (재전송 곤란 시) |

# 3. 주요 ARQ(Automatic Repeat Request) 방식의 개념 및 수행 절차

## 가. 정지-대기(Stop-and-Wait) ARQ
- **개념**: 한 개의 프레임을 전송한 후, 수신측으로부터 **ACK(긍정 응답)**를 받을 때까지 다음 프레임 전송을 중단하고 대기하는 방식
- **수행 절차**:
    1.  송신측: 프레임 전송 후 **타이머(Timer)** 작동
    2.  수신측: 오류가 없으면 ACK 전송, 오류 발생 시 NAK 전송 또는 무응답
    3.  송신측: ACK 수신 시 다음 프레임 전송 / NAK 수신 또는 타임아웃 발생 시 기존 프레임 재전송

## 나. Go-Back-N ARQ
- **개념**: 여러 개의 프레임을 **윈도우(Window)** 크기만큼 연속해서 전송하고, 오류 발생 시 오류가 발생한 프레임부터 **그 이후의 모든 프레임**을 다시 전송하는 방식
- **수행 절차**:
    1.  송신측: 윈도우 크기 내에서 프레임 번호를 부여하여 연속 전송
    2.  수신측: 오류 발생 프레임 발견 시 해당 번호 이후의 모든 프레임 폐기 및 NAK 전송
    3.  송신측: NAK를 받은 프레임 번호부터 현재까지 전송된 모든 프레임을 재전송

# 4. Stop-and-Wait ARQ와 Go-Back-N ARQ 비교

| 비교 항목 | 정지-대기 (Stop-and-Wait) | Go-Back-N (GBN) |
|---|---|---|
| **전송 효율** | 낮음 (대기 시간 발생) | 높음 (연속 전송 가능) |
| **윈도우 크기** | 송신 1, 수신 1 | 송신 N, 수신 1 |
| **구현 난이도** | 단순 (가장 기초적) | 중간 (버퍼 관리 필요) |
| **오류 발생 시** | 해당 프레임 1개만 재전송 | 오류 발생 이후 모든 프레임 재전송 |
| **주요 특징** | 전송 지연이 큰 환경에 취약 | 중복 재전송으로 인한 대역폭 낭비 가능 |

# 5. 기술사적 제언 및 발전 방향

## 가. Selective Repeat ARQ로의 발전
- Go-Back-N의 단점인 중복 재전송을 해결하기 위해, 오류가 발생한 **특정 프레임만 선택적으로 재전송**하는 Selective Repeat 방식이 현대 통신에서 널리 쓰임

## 나. 하이브리드 ARQ (H-ARQ)
- 4G/5G 이동통신에서는 FEC(오류 정정)와 ARQ(재전송)를 결합한 **H-ARQ**를 사용하여 전송 효율과 신뢰성을 동시에 극대화함

> [!tip] **기술사 인사이트**
> 오류 제어는 전송 매체의 **BER(Bit Error Rate)** 환경에 따라 최적의 전략을 선택해야 합니다. 대역폭 효율이 중요한 현대 통신에서는 단순 재전송(BEC)보다는 **Soft-Combining** 기술이 포함된 H-ARQ를 심도 있게 이해하는 것이 고득점의 포인트입니다.
